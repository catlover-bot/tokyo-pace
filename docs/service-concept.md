# サービスコンセプト

動的経路は「安全な経路」ではなく、OpenStreetMap属性に基づく比較候補です。標準、階段回避要求、車いすプロファイルをTOKYO PACEの決定的な条件負担スコアで再評価します。車いすで通れる、必ず通れる、施設を利用・休憩できるとは断定せず、現地の工事、段差、路面、入場条件、営業時間の確認を促します。

## 3候補を選ぶための説明

TOKYO PACEは3候補を順位だけで示さず、推奨理由、標準候補との距離・時間・休憩空白の差、利点、注意点、満たさなかった条件を並べます。推奨は既存の決定的な条件負担スコアで1位の候補であり、AIが本番時に文章や順位を生成しません。理由コードと評価値から、根拠がある説明だけを固定順で生成します。

標準候補があれば差分基準とし、なければ最短候補を決定的に選びます。選択した候補は比較表・カード・地図で同期します。色に加えて順位、文字バッジ、実線・破線・点線、選択中表示を使い、キーボードや読み上げでも比較できる構造を目指します。

条件負担スコアは低いほど、設定した歩行条件に近いことを表しますが、最適性、安全性、通行可能性を示す値ではありません。条件未達の候補も、超過時間や不足情報を明示して比較対象に残します。固定デモは新宿駅西口から東京都庁までの説明用データであり、動的なopenrouteservice候補とは別の表示・出典にします。

休憩候補は`confirmed`（現地確認・座席明示）、`supported`（公式掲載と人による現地情報の複数根拠）、`possible`（施設の存在のみ）、`estimated`（デモ推定）を分離します。公共施設やバリアフリー掲載施設を自動的に休憩可能とは判定しません。

現地確認は「公式掲載施設を確認済みに見せる」ためではなく、未確認事項を人が一つずつ埋めるためのワークフローです。確認日時、非個人の確認主体ID、現地確認method、一般利用、座席または明示的な休憩空間が揃った場合だけconfirmedとし、複数根拠がある不完全な確認をsupported、根拠不足をpossibleに残します。否定値や不明値を都合よく肯定へ読み替えません。

現地確認支援画面は候補の読み取りとCSVテンプレート取得だけを提供します。匿名ユーザーが確認結果を書き込むAPIや写真アップロードは設けず、個人名・連絡先を収集しない運用を前提にします。候補順位は「現地確認する価値」の理論順位であり、その施設が休憩可能、安全、営業中、設置可能であることを示しません。

確認後の効果は、厳格な休憩候補を入れる前後の最長空白、最大連続歩行、成立判定、被覆率として並べます。実確認がなければ前後は同値であり、改善を表示しません。possibleやestimatedだけで「移動可能」「休憩できる」と断定しません。

距離は施設から折れ線までの推定直線距離、またはデモ総距離へ正規化したルート沿い推定距離です。実際の徒歩経路、営業時間、入館条件、着席、給水地点の座席、安全性は保証しません。追加休憩地点は都市設備配置検討の理論値です。

選択経路から生成するCSV/GeoJSONは、TOKYO PACEの計算を検証・再利用するための派生分析データです。経路のOpenStreetMap/ODbL、東京都・新宿区のCC BY、現地確認、理論分析をFeature単位で分離し、出典と警告を残します。分析ファイルの存在も、経路や施設の利用可能性を保証するものではありません。

TOKYO PACE は、長時間続けて歩くことが難しい人が、休憩候補・トイレ候補・坂道の情報を比較し、「最後まで歩き切れる可能性の高い経路」を自分で選ぶための支援サービスです。最短経路の代替ではなく、判断材料を増やすことを目的とします。

## 中心指標

経路を休憩機会ごとの連続歩行区間に分け、利用者が設定した「休まずに歩ける時間」と比較します。最大連続歩行時間、次の休憩候補までの最長距離（最長休憩空白）、上限超過時間を区間データから導出し、移動を継続できる条件かを平易な日本語で示します。この判定は経路全体の安全保証ではなく、利用者が比較するための一つの判断材料です。

## MVP の前提

- 動的検索は新宿駅・東京都庁・新宿中央公園周辺の限定bboxを対象とし、固定デモだけが新宿駅西口から東京都庁までです。
- 動的経路はopenrouteservice / OpenStreetMap由来、地点・設備属性は公式オープンデータ、固定経路と一部の休憩地点は推定デモデータです。出典区分を混同しません。
- 区間距離・区間歩行時間と、それらから導出する移動継続可能性は、動的経路またはデモ値からの推定です。現地の実測値ではありません。
- 公衆トイレと車椅子使用者対応情報には新宿区・東京都福祉局の公式オープンデータを使用します。公式掲載は現在利用可能であることや、そこまで通行可能であることを保証しません。
- 安全性、バリアフリー性、設備の利用可否を保証しません。
- 高齢者を主対象に、大きな操作領域、文字情報による凡例、地図外の一覧を提供します。

## 公式情報と推定情報

公式トイレは利用保証や経由地点ではなく、「ルート周辺に公式掲載情報がある候補地点」と表現します。同一建物・同一座標に複数の設備レコードがあっても候補地点数を過大評価せず、詳細から原レコードと出典を確認できるようにします。

公衆トイレ候補と、公共施設内・鉄道駅内の車椅子使用者対応設備情報は評価上も分離します。既定条件は250m以内の公衆トイレ候補だけで成立します。ルート上で公衆トイレ候補の間隔が最も長い区間を「最長トイレ空白」として示しますが、これはデモ折れ線上の推定であり、実際の徒歩距離や利用可否を保証しません。

「ルートから候補まで」は折れ線への推定直線距離です。「ルート上の空白」は折れ線上の位置比率を画面表示のデモ総距離へ換算したルート沿い推定距離です。この2つは異なる尺度で、実際の道路ネットワーク上の徒歩距離ではありません。本番用ルーティングでは道路区間ごとの距離に置き換える予定です。

地図では公式トイレを「公式オープンデータ」、休憩候補と経路を「推定デモデータ」として分けます。車椅子使用者対応情報が空欄なら「不明」とし、非対応とは扱いません。ルート近傍判定は直線距離の推定であり、実際の迂回、工事、故障、営業時間、入場制限、経路全体の車椅子通行可能性を判断しません。
